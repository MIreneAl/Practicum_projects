# Тестирование API Яндекс.Прилавка (бэкенд)

## Описание проекта
**Цель:** Проверка функциональности, надёжности и обработки ошибок API бэкенда сервиса Яндекс.Прилавок.  
**Объект тестирования:** API версия 3.1.1  
**Время проведения работ:** 12.08.2025 – 19.08.2025  
**Исполнитель:** Маркова Ирина

## Стек технологий и инструменты
- **Тестирование API:** Postman (ручное тестирование, коллекции)
- **Клиент:** ОС Windows 10, браузер Opera GX версия 116.0.5366.157
- **Сервер:** Яндекс.Прилавок (удалённый стенд)
- **Управление тестовой документацией:** Google Sheets, Google Docs
- **Баг-трекинг:** Google Sheets (таблица со скриншотами) + PDF-отчёт в репозитории
- **Техники тест-дизайна:** классы эквивалентности, анализ граничных значений, негативное тестирование

## Результаты тестирования
- **Всего обнаружено дефектов:** 59
  - Критический приоритет: 6
  - Средний приоритет: 35
  - Низкий приоритет: 18

### Ключевые дефекты (критические)
- **D-12:** Удаление обязательного поля вызывает ошибку 500
- **К-36:** Возможность создания корзины вне часов работы склада
- **КУ-53:** Некорректное удаление существующей корзины
- **МН-54, МН-55:** Отсутствие валидации данных перед отправкой на сервер + некорректное отображение диалогового окна

*Полное описание багов приведено в PDF-файле `Bug_Reports/Bug_Reports.pdf`. Для просмотра скриншотов используйте [таблицу баг-репортов](https://docs.google.com/spreadsheets/d/10w52Yr5PBRbw5vCpvPvJHXkhtIlP225tEmT6nVDPCn8/edit?usp=sharing).*

## Выводы и рекомендации
- **Версия не готова к релизу.** Основные функциональности (создание и удаление корзины) содержат блокирующие дефекты, система обработки ошибок реализована не полностью.
- **Рекомендации:**
  1. Устранить критические дефекты.
  2. Доработать валидацию на бэкенде (возвращать понятные ошибки 400).
  3. Реализовать корректную обработку ответов сервера на фронтенде.
  4. Уточнить требования в части «серых зон» (формат времени доставки, граничные значения веса/количества и т.д.).
  5. Провести регрессионное тестирование после исправлений.

## Серые зоны (неоднозначности требований)
- Диапазон времени доставки, формат, учёт часового пояса.
- Допустимость нулевых значений веса и количества.
- Обязательность полей «время доставки», «вес», «количество».
- Максимальные и минимальные значения для веса и количества.
- Поведение при запросе несуществующей или пустой корзины.
- Максимальное количество наборов в корзине, реакция на переполнение.
- Коды ответов и сообщения при неудачном удалении корзины.
- Обработка недопустимых форматов времени.
- Логика выбора службы доставки при одинаковых критериях.
