# Тестирование функциональности «Сделать заказ» (веб-приложение для аренды самокатов)

## Описание проекта
**Цель:** Проверка ключевого пользовательского сценария «Сделать заказ» в веб-приложении.  
Тестирование проводилось на стенде:  
[https://90090cb5-ebc1-4ce2-ba6a-ca363883fecb.serverhub.praktikum-services.ru/](https://90090cb5-ebc1-4ce2-ba6a-ca363883fecb.serverhub.praktikum-services.ru/)

**Что было протестировано:**
- **Вёрстка формы «Для кого самокат»** – адаптивность, отображение элементов на разрешениях 1920×1080 и 1280×720.
- **Логика обеих форм** («Для кого самокат» и «Про аренду») – позитивные и негативные сценарии, валидация полей, переходы между шагами, оформление заказа.
- **Шапка сайта** – проверка отображения логотипа, текстовой надписи «Учебный тренажер», кнопок «Заказать» и «Статус заказа», поля ввода номера заказа; стили (цвета, шрифты, hover-эффекты), навигационные переходы (на страницу Дзен, на главную, к полю статуса), отсутствие наложений элементов (тесты Ш‑1 … Ш‑25).
- **Работа с куки** – отображение баннера с информацией о куки, его скрытие при нажатии на кнопку «да все привыкли», сохранение состояния на время сессии (тесты К‑1 … К‑4).

**Стек технологий и инструменты:**
- Ручное тестирование (functional, UI/UX).
- ОС: Windows 11 Pro.
- Браузеры: Google Chrome 142.0.7444.175, Яндекс.Браузер 25.10.1.1136.
- Разрешения экрана: 1920×1080, 1280×720.
- Управление тест-документацией: Google Sheets.
- Баг-трекинг: дефекты зафиксированы в чек-листе (Google Sheets) с указанием приоритетов.
- Техники тест-дизайна: классы эквивалентности, анализ граничных значений, позитивное/негативное тестирование.

## Результаты тестирования
- **Всего проверок:** 221  
- **Успешно:** 172 (77,8%)  
- **Неуспешно:** 49 (22,2%)  

### Ключевые дефекты

**Блокирующий:**
- БР-18: Кнопка «Да» в окне подтверждения заказа не кликабельна

**Критичные:**
- БР-7: Текст всплывающего окна не соответствует макету
- БР-8: Система позволяет выбрать дату в прошлом

**Среднего приоритета (объединены):**
- БР-1, БР-2, БР-5, БР-9, БР-10, БР-11, БР-12, БР-13, БР-14, БР-15, БР-16, БР-17

**Низкого приоритета (объединены):**
- БР-3, БР-4, БР-6, БР-19, БР-20

*Полное описание всех дефектов (шаги воспроизведения, ожидаемый и фактический результат) приведено в файле `Bug_Reports.pdf`.*

### Выводы
- Функциональность **не готова к релизу** из-за блокирующего дефекта БР-18 (невозможно завершить заказ).
- Необходимо исправить критические ошибки: БР-7 (несоответствие макету), БР-8 (некорректная валидация даты).
- В ходе тестирования выявлены «серые зоны» – отсутствие требований к логике всплывающих окон, правилам генерации номера заказа, поведению при отмене заказа.
- Рекомендуется:
  - Уточнить требования к модальным окнам и полю ввода телефона (маска ввода).
  - Разрешить ввод дефиса и пробела в поле «Фамилия».
  - Исключить возможность ручного ввода даты доставки, разрешив только выбор из календаря, и запретить выбор прошедших дат.
